GuardDuty and security monitoring

Amazon GuardDuty is a threat detection service that analyzes signals such as CloudTrail events, DNS activity, VPC Flow Logs, EKS audit logs, and other telemetry to identify suspicious behavior.

GuardDuty can detect patterns such as credential misuse, unusual API calls, reconnaissance, communication with suspicious hosts, anomalous data access, and signs of compromised compute resources.

GuardDuty should be enabled across accounts and regions. In multi-account environments, use a delegated administrator so findings can be managed centrally.

Findings should not just be collected; they need triage and response. High-severity findings should create alerts, tickets, or automated containment workflows depending on the environment.

Useful response actions include disabling exposed credentials, isolating instances, blocking malicious network paths, reviewing CloudTrail activity, rotating secrets, and preserving evidence.

GuardDuty is stronger when paired with other services. Security Hub can aggregate findings, EventBridge can trigger workflows, CloudTrail provides investigation detail, and IAM Access Analyzer helps identify unintended access.

Common monitoring gaps include enabling detection in only one account, failing to route findings to responders, ignoring low-and-medium findings that repeat over time, and not testing whether alerts actually reach the team.

GuardDuty does not replace secure configuration. It detects suspicious behavior, but teams still need least privilege, logging, patching, encryption, network controls, and incident response procedures.
